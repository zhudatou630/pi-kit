# 决策记录模板

每次 grill 达成共识后，在 `.pi/grill/` 下新建一个文件（命名见 SKILL.md「落盘」），按此格式填写。保持简洁——这是给未来的自己看的决策记录，不是报告。

```markdown
# <主题>

**时间**：YYYY-MM-DD HH:MM
**背景**：<一句话，要解决什么>

**关键决策**：
- <决策1> → <选定项>（备选：<...>）— 理由：<...>
- <决策2> → <选定项>（备选：<...>）— 理由：<...>

**待定/未决**：<明确标出还没定的，及为什么搁置>

**事实依据**：<拷问中 agent 查到的事实，带来源：文件路径 / URL / 计算结果>

**下一步**：<共识后要执行什么>
```

要点：
- 决策只记"定了什么 + 为什么 + 备选"，不记拷问过程。
- 待定项必须显式列出——隐性假设就是从这里漏掉的。
- 事实依据带来源，方便日后复核。
